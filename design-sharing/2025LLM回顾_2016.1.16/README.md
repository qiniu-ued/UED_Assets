# Andrej Karpathy: 2025 LLM Year in Review
**解读：定义 AI 新时代的 6 大范式转移**

> **摘要**：
> 2025 年是 LLM 发展的分水岭。如果说之前我们是在训练模型“模仿”人类（System 1），那么 2025 年则是模型学会“思考”和“推理”（System 2）的元年。
> 
> 本文总结了 Andrej Karpathy 提出的定义这一年的 **6 大范式转移 (6 Paradigm Shifts)**。

---

## 1. 训练范式的终极进化：RLVR
**Reinforcement Learning from Verifiable Rewards (基于可验证奖励的强化学习)**

在 2025 年之前，标准的 LLM 训练流水线是：`Pre-training` $\rightarrow$ `SFT` $\rightarrow$ `RLHF`。
而现在，**RLVR** 成为了新的标准第四阶段。

* **从“主观评价”到“客观真值”**：
    * **RLHF (旧)**：依赖人类的主观打分（"这就话写得不错"），难以扩展且昂贵。
    * **RLVR (新)**：依赖客观结果（"代码跑通了吗？"、"数学题做对了吗？"）。
* **思考时间 (Thinking Time)**：
    * 通过 RLVR，模型学会了在输出答案前进行“思维链 (Chain of Thought)”推理。
    * 算力重心从“训练端”向“推理端”发生巨大迁移 (**Test-time Compute**)。模型可以通过“多想一会儿”来线性提升智力。

## 2. 智能的新形态：Ghosts vs. Animals
**幽灵 vs. 动物 (锯齿状智能 / Jagged Intelligence)**

Karpathy 用了一个非常形象的比喻来描述当前的 AI：

* **动物 (Animals)**：生物进化是平衡的。一只聪明的动物通常也具备敏锐的感知和运动能力。
* **幽灵 (Ghosts)**：LLM 像是我们通过咒语“召唤”出来的幽灵。
    * 它们是**严重的偏科生**：在数学、编程等有“验证信号”的领域（RLVR 覆盖区），它们表现出超越人类的天才。
    * **锯齿状智能**：在常识、物理世界交互等没有明确奖励信号的领域，它们依然非常脆弱，甚至表现得像个“白痴”。

## 3. 应用层的新物种：Cursor
**垂直整合的 AI 原生编辑器**

* **IDE 的消亡**：传统的 VS Code 插件模式（仅做 Copilot 补全）已经过时。
* **编辑器即 Agent**：Cursor 展示了 LLM 如何接管整个编码环境。
    * 它不仅仅是补全代码，而是理解整个项目上下文。
    * 它可以管理文件差异（Diffs）、并在多个文件中主动修改代码。
* 这标志着 AI 应用从“聊天窗口”向“深度集成工具”的转变。

## 4. 部署架构的修正：Claude Code
**AI 应该住在 Localhost，而不是云端**

Karpathy 认为 OpenAI 在 Agent 路线上的方向曾一度走偏（试图在云端容器中运行 Agent Swarms），而 Anthropic 通过 **Claude Code** 找对了方向：

* **Local Spirit (本地精灵)**：AI 不应该只是网页上的一个 Chatbot，它应该是一个住在你终端（Terminal）里的“幽灵”。
* **权限与环境**：它能直接访问你的文件系统、运行本地工具、读取你的 Git 仓库。
* **CLI 形式**：通过命令行界面，AI 成为了你计算机的原生一部分，而不是一个远程的 API。

## 5. 编程方式的革命：Vibe Coding
**氛围编程**

> *"I just manage the vibes."*

编程的定义被彻底改写：

* **传统编程**：编写具体的语法、逻辑、分号。
* **Vibe Coding**：开发者通过自然语言描述意图（Intents）和氛围（Vibes）。
* **一次性软件 (Disposable Software)**：因为写代码变得极度廉价，我们开始编写大量“用完即弃”的小型软件。开发者从“泥瓦匠”变成了“项目经理”，主要工作是 Review 代码和调整需求。

## 6. 交互界面的未来：Nano Banana
**从文本流到生成式 UI (Generative UI)**

*注："Nano Banana" 是 Karpathy 用来指代多模态生成式 UI 的特定代号/案例。*

* **超越 Text-in, Text-out**：与 LLM 的交互不再局限于黑底白字的控制台。
* **Generative UI**：模型能够根据内容实时生成图形界面（GUI）。
    * 如果你问它股票数据，它直接生成一个交互式图表组件。
    * 如果你问它食谱，它生成一个可视化的卡片。
* **多模态输出**：未来的计算机会根据语境，动态生成最适合当前任务的用户界面，而不是预先写死的界面。

---

## 💡 总结

2025 年并不是 AGI 降临的一年，而是 AI 从**“概率模仿者”**蜕变为**“逻辑推理者”**的一年。

* **对于开发者**：学会 **Vibe Coding**，习惯与 **Localhost Agent** 共存。
* **对于行业**：RLVR 打开了新的大门，只要能定义“奖励函数”的领域，AI 都将迎来指数级进化。
